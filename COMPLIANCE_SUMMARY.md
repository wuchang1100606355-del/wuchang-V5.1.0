# 五常系統合規摘要

## 📋 核心合規聲明

**本系統除法律規範須依法揭露及政府公示資訊中公開揭露之外無可供識別之個資，應屬合規。**

**本系統可究責之自然人不在隱私權保護規範內。**

**本系統及AI程序設計之可究責自然人不在隱私權保護規範內。**

**本系統經授權之獨立管理權限自然人不在隱私權保護規範內，此兩種自然人（本系統及AI程序設計之可究責自然人、本系統經授權之獨立管理權限自然人）除姓名外其餘不得公開揭露，但須紀錄於本系統之硬編碼。**

## ✅ 合規保證

### 1. 系統設計層面

- ✅ **不收集個資**：系統不主動收集、儲存或處理個人可識別資訊（法律規範須依法揭露及政府公示資訊中公開揭露者除外）
- ✅ **不處理個資**：所有資料處理流程不涉及個人可識別資訊（法律規範須依法揭露及政府公示資訊中公開揭露者除外）
- ✅ **不傳輸個資**：系統間通訊不包含個人可識別資訊（法律規範須依法揭露及政府公示資訊中公開揭露者除外）
- ✅ **稽核不記錄個資**：所有稽核記錄不包含個人可識別資訊（法律規範須依法揭露及政府公示資訊中公開揭露者除外）

### 2. 系統感知層面

- ✅ **感知功能無個資**：所有系統感知讀數不包含個人可識別資訊
- ✅ **記憶儲存無個資**：系統神經網路記憶不包含個人可識別資訊
- ✅ **API 回應無個資**：所有感知 API 回應不包含個人可識別資訊

### 3. 預留功能處理

- ✅ **個資功能未啟用**：所有個資相關的預留功能目前未啟用
- ✅ **返回合規聲明**：個資相關 API 返回合規聲明而非實際資料
- ✅ **不影響合規性**：預留功能本身不影響「無個資」合規聲明

## 📊 資料分類

### ✅ 允許的資料類型（非個資）

- 系統狀態資料（服務狀態、資源使用、網路狀態）
- 匿名統計資料（人口結構統計，不包含個人識別資訊）
- 商業生態資料（商家資訊，由商家自行提供並同意揭露）
- 地理空間資料（座標、街道、區域邊界）
- 系統配置資料（環境變數、設定檔、規則檔）
- 稽核事件資料（操作類型、時間戳、結果，不含個人識別資訊）
- 帳號權限狀態（僅權限資訊，不含個人資料）

### ❌ 不處理的資料類型（個資）

- 姓名、身份證字號、護照號碼（法律規範須依法揭露及政府公示資訊中公開揭露者除外）
- 電話號碼、電子郵件地址（除非為公開的商家資訊或法律規範須依法揭露及政府公示資訊中公開揭露者）
- 地址（除非為商家公開資訊或法律規範須依法揭露及政府公示資訊中公開揭露者）
- 生日、年齡（除非為匿名統計或法律規範須依法揭露及政府公示資訊中公開揭露者）
- 生物特徵資料（法律規範須依法揭露及政府公示資訊中公開揭露者除外）
- 財務資訊（除非為公開的商家資訊或法律規範須依法揭露及政府公示資訊中公開揭露者）

### ⚖️ 例外情況（法律規範與政府公示）

以下情況屬於例外，不違反合規聲明：

1. **法律規範須依法揭露**：依法律規定必須揭露的資訊（如公司登記、商業登記、食安究責資料等）
2. **政府公示資訊中公開揭露**：政府機關已公開的資訊（如公司登記資訊、商業登記資訊、公開招標資訊等）
3. **可究責之自然人不在隱私權保護規範內**：
   - 公司/組織的負責人資訊（依法律規定必須揭露）
   - 合規名冊中的聯絡窗口與可究責自然人（如食安究責資料）
   - 法律要求必須揭露的負責人、代表人、聯絡人等資訊
   - 這些資訊屬於「可究責」性質，不在一般隱私權保護規範範圍內

4. **本系統及AI程序設計之可究責自然人不在隱私權保護規範內**：
   - 系統設計者、開發者、維護者等可究責自然人
   - AI程序設計者、開發者、維護者等可究責自然人
   - 系統管理員、技術負責人等可究責自然人
   - 這些資訊屬於「可究責」性質，不在一般隱私權保護規範範圍內

5. **本系統經授權之獨立管理權限自然人不在隱私權保護規範內**：
   - 經授權具有獨立管理權限的自然人（如系統管理員、權限管理員等）
   - 具有系統獨立管理權限的授權人員
   - 這些資訊屬於「可究責」性質，不在一般隱私權保護規範範圍內

6. **此兩種自然人的資訊處理規範**：
   - **本系統及AI程序設計之可究責自然人**
   - **本系統經授權之獨立管理權限自然人**
   - **除姓名外其餘不得公開揭露**：姓名可以公開，但其他資訊（如聯絡方式、職務詳情、技術細節等）不得公開揭露
   - **但須紀錄於本系統之硬編碼**：所有相關資訊必須記錄在系統的硬編碼中（如配置檔、程式碼註解、系統文件等），以確保責任歸屬和系統管理

## 🔐 合規機制

### 自動過濾

所有資料輸出都會自動過濾個資：

```python
# 感知讀數不包含個資
value = {
    "admin_email": "admin@wuchang.life",  # 僅帳號識別，非個資
    "is_admin": True,  # 權限狀態，非個資
    "username": "admin",  # 系統帳號，非個資
    # 不包含：姓名、電話、地址等個資
}
```

### 預留功能處理

所有個資相關的預留功能在「無個資」模式下：

- 返回合規聲明而非實際資料
- 標記為未啟用
- 不執行任何個資處理操作
