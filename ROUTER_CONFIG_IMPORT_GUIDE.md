# 華碩路由器設定檔案匯入指南

## 📋 設定檔案資訊

- **檔案名稱**: `Settings_RT-BE86U.CFG`
- **檔案位置**: `J:\我的雲端硬碟\Bound\Downloads\Settings_RT-BE86U.CFG`
- **路由器型號**: ASUS RT-BE86U

## 🔧 匯入方法

### 方法一：通過 Web 介面匯入（推薦）

#### 步驟

1. **連接到路由器管理介面**
   - 本地連線：`https://192.168.50.84:8443`
   - 遠端連線：`https://220.135.21.74:8443`
   - 或使用 DDNS：`https://coffeeLofe.asuscomm.com:8443`

2. **登入路由器**
   - 使用管理員帳號和密碼登入

3. **進入設定備份/還原頁面**
   - 路徑：**系統管理** → **設定** → **備份/還原設定**
   - 或：**Administration** → **Settings** → **Backup/Restore Settings**

4. **選擇還原設定**
   - 找到「還原設定」或「Restore Settings」區塊
   - 點擊「選擇檔案」或「Browse」按鈕

5. **選擇設定檔案**
   - 瀏覽到：`J:\我的雲端硬碟\Bound\Downloads\`
   - 選擇檔案：`Settings_RT-BE86U.CFG`

6. **上傳並還原**
   - 點擊「上傳」或「Restore」按鈕
   - 確認還原操作（可能會警告此操作會覆蓋現有設定）

7. **等待路由器重新啟動**
   - 路由器會自動重新啟動以套用新設定
   - 重新啟動過程可能需要 1-3 分鐘
   - **重要**：重新啟動期間網路會中斷

### 方法二：通過 SSH 匯入（進階）

如果路由器已啟用 SSH，可以使用以下方法：

#### 步驟

1. **連接到路由器 SSH**
   ```powershell
   ssh router-65433
   # 或
   ssh -i "$env:USERPROFILE\.ssh\id_ed25519_router" -p 65433 admin@220.135.21.74
   ```

2. **上傳設定檔案到路由器**
   ```powershell
   # 使用 SCP 上傳檔案
   scp -i "$env:USERPROFILE\.ssh\id_ed25519_router" -P 65433 `
       "J:\我的雲端硬碟\Bound\Downloads\Settings_RT-BE86U.CFG" `
       admin@220.135.21.74:/tmp/Settings_RT-BE86U.CFG
   ```

3. **在路由器上還原設定**
   ```bash
   # SSH 到路由器後執行
   cd /tmp
   nvram restore Settings_RT-BE86U.CFG
   nvram commit
   reboot
   ```

   **注意**：此方法需要路由器支援 `nvram restore` 命令，且操作風險較高。

### 方法三：使用 Python 腳本（實驗性）

已提供 `import_router_config.py` 腳本，但需要正確的 API 端點。

```powershell
python import_router_config.py
```

**注意**：此方法可能需要根據實際路由器 API 調整。

## ⚠️ 重要注意事項

### 匯入前準備

1. **備份現有設定**
   - 在匯入新設定前，務必先備份當前路由器設定
   - 路徑：**系統管理** → **設定** → **備份/還原設定** → **備份設定**

2. **確認設定檔案**
   - 確認設定檔案是針對 RT-BE86U 型號
   - 確認設定檔案未損壞
   - 建議先檢查檔案大小和修改時間

3. **網路連線**
   - 確保有穩定的網路連線
   - 建議使用有線連線而非無線連線
   - 準備好路由器重新啟動後可能需要重新連線

### 匯入後檢查

1. **網路連線**
   - 確認網路連線正常
   - 檢查 IP 設定是否正確

2. **Wi-Fi 設定**
   - 確認 Wi-Fi SSID 和密碼
   - 測試無線連線

3. **端口轉發**
   - 檢查端口轉發規則是否正確
   - 確認 SSH 端口 65433 轉發設定

4. **DDNS 設定**
   - 確認 DDNS 服務是否正常運作
   - 檢查 `coffeeLofe.asuscomm.com` 是否正常解析

5. **防火牆規則**
   - 確認防火牆規則是否正確
   - 檢查必要的端口是否開放

## 🔍 故障排除

### 問題：無法連接到路由器管理介面

**解決方案**：
- 確認路由器 IP 地址正確
- 檢查網路連線
- 嘗試使用不同的端口（443, 8080, 8444）
- 確認防火牆未阻擋連線

### 問題：設定檔案上傳失敗

**解決方案**：
- 確認檔案格式正確（.CFG 檔案）
- 檢查檔案大小（不應超過路由器限制）
- 嘗試使用不同的瀏覽器
- 清除瀏覽器快取和 Cookie

### 問題：匯入後路由器無法正常運作

**解決方案**：
1. **重置路由器**
   - 按住路由器背面的 Reset 按鈕 10 秒
   - 路由器會恢復到出廠預設值

2. **使用備份設定還原**
   - 如果有備份，可以重新匯入備份設定

3. **手動重新設定**
   - 如果沒有備份，需要手動重新設定路由器

### 問題：匯入後網路設定遺失

**解決方案**：
- 檢查 IP 設定（DHCP 或靜態 IP）
- 確認 DNS 設定
- 檢查 VLAN 設定（如果使用）

## 📝 設定檔案格式說明

華碩路由器的 `.CFG` 設定檔案通常是：
- **格式**：可能是加密或編碼的二進制格式
- **內容**：包含所有路由器設定（網路、Wi-Fi、防火牆、端口轉發等）
- **版本**：不同韌體版本的設定檔案可能不相容

## 🔒 安全建議

1. **保護設定檔案**
   - 設定檔案可能包含敏感資訊（密碼、金鑰等）
   - 不要將設定檔案分享給他人
   - 儲存在安全的位置

2. **定期備份**
   - 定期備份路由器設定
   - 在重大變更前務必備份

3. **驗證設定**
   - 匯入後仔細檢查所有設定
   - 確認安全設定（防火牆、訪問控制等）正確
